## PR description

### Summary
- **iOS terminal: snapshot-mode rewrite** — switches from raw PTY stream to tmux `capture-pane` snapshots, eliminating the cursor-residue bug (no client-side terminal state machine)
- **New backend endpoints** for iOS to consume: HTTP capture/send + WS snapshot push with FNV64 dedup
- **Electron mkcert TLS support** — enables iPhone to connect via `wss://` through frps tunnel with self-signed cert

### Backend (`cmd/gui/server.go`)
- `GET /api/sessions/{name}/capture` → `tmux capture-pane` (optional `?history=N` for scrollback)
- `POST /api/sessions/{name}/send` → `tmux send-keys` (reuses existing `sendToTmuxSession`)
- `WS /ws/snapshot/{name}` → pushes JSON frames `{type, content, ts}`, 100ms throttle with content-hash dedup

### iOS
- `TerminalViewModel`: snapshot replace (no `\r` merge), HTTP POST for send (server handles CR), 30s ping + exponential backoff reconnect, foreground auto-reconnect via `makroReconnect` notification, `refresh()` for manual pull
- `SessionsView`: refresh button in toolbar with spinning icon
- `Config`: new URL helpers (`snapshotWSURL`, `sessionSendURL`, `sessionCaptureURL`, `bearerToken`)

### Electron
- `main.js`: detects `certs/127.0.0.1+2.pem`, passes `--tls-cert/--tls-key`, `waitForBackend` uses https client, renderer trusts self-signed cert via `setCertificateVerifyProc`
- `preload.js`, `frontend/src/main.js`, `package.json`: accompanying TLS plumbing
